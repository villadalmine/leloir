Leloir

An open-source agentic ops platform where any AI agent can orchestrate any other to resolve operational incidents — with unified cost, audit, approvals, and security.

In honor of Luis Federico Leloir — Nobel Prize in Chemistry, 1970 · Discovered how the body metabolizes complex sugars · Mentored by Houssay, mentor to many · A symbol of investigation and lineage.

The name is no accident. Luis Federico Leloir spent decades tracing how raw material — sugar — becomes the fuel that runs every cell. He discovered the nucleotide sugars: the intermediate compounds nobody had seen, the missing steps in a metabolism everyone assumed was already understood. He was a student of Bernardo Houssay, who in turn taught a generation of researchers. The lineage mattered as much as the work.

This platform is named after him because that is what it does. When an alert fires — a vague, surface-level symptom — Leloir's agents trace the metabolism of your systems backward to the root cause. And like Leloir himself, they don't work alone: a primary agent calls on specialists, a generalist consults a domain expert, a junior reaches for a senior. The lineage matters as much as the work.

Section II · A worked example

An incident, traced.

A real production scenario: PostgreSQL connection pool exhaustion in a tenant namespace. Watch how a primary agent reasons, then calls a specialist, then closes the case. Total wall-clock time: roughly forty seconds. Total LLM cost: $0.34.

01
Alertmanager · 14:32:01
Fires PostgresConnectionPoolHigh for tenant-alpha. Webhook hits Leloir's ingestion endpoint.
labels: { tenant: alpha, severity: critical, type: database }
02
Routing Engine · 14:32:01
Matches AlertRoute prod-database-incidents. Selects agent holmes-prod. Grants tools from postgres-mcp + prometheus-mcp + kubernetes-mcp. Authorizes A2A team [security-specialist, dba-specialist].
03
HolmesGPT · 14:32:03 → 14:32:18
Investigates. Queries postgres.query → 423 of 500 connections in use. Queries kubernetes.logs → "max_connections exceeded" errors. Notes a recent deployment of payment-service v2.3.
3 tool calls · 2 LLM calls · $0.18
A2A
Holmes invokes Security-Specialist · 14:32:19
"Is the connection pattern compromised, or is it organic?"
Caller hint: max_cost_usd 2.00. Effective budget: $1.42 (parent has $4.66 remaining; tenant cap $50; route cap $5; min wins).
→
Security-Specialist · 14:32:19 → 14:32:34
Queries network.flow_logs → 87% of connections originate from a single pod IP. Queries kubernetes.describe_pod → matches v2.3, deployed 47 minutes ago. Concludes: connection leak in payment-service v2.3, not an attack.
2 tool calls · 2 LLM calls · $0.16 · returns to Holmes
04
HolmesGPT · 14:32:35 → 14:32:41
Integrates the security verdict. Drafts the recommendation: roll back payment-service to v2.2; open ticket for the leak in v2.3; pool will recover within 2 minutes once new connections stop opening.
final LLM call · $0.00 (cached) · investigation total: $0.34
05
Notification Fan-out · 14:32:41
Posts an Adaptive Card to #sre-tenant-alpha in Teams. Hash of the audit chain anchored to S3 Object Lock 5 minutes later.
